# Contributing

Thank you for considering a contribution to this project.

## How to contribute

1. Fork the repository and create a branch from `main`.
2. Make your changes. Add or update tests for any new behavior.
3. Run `dotnet build` and `dotnet test` to verify everything passes.
4. Open a pull request with a clear description of what you changed and why.

## Code style

- Use the project's existing conventions (nullable enabled, warnings as errors).
- Keep methods short and focused.
- Prefer composition over inheritance.

## Reporting issues

Open a GitHub issue. Include steps to reproduce, expected behavior, and actual behavior.
